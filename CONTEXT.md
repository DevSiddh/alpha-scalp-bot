# Alpha-Scalp Bot — Project Context

> **Last updated:** 2026-03-14  
> **Status:** Active development — Phase 1 complete, Phase 2 (LLM weight optimizer) integrated

---

## 1. Overview

Alpha-Scalp Bot is a fully automated cryptocurrency futures scalping + swing trading system that runs on Binance Futures via the CCXT library. It uses a multi-signal voting architecture where independent technical signals vote on direction, are weighted and aggregated into a composite score, and trade only when the score exceeds a confidence threshold.

**Exchange:** Binance Futures (USDT-marginal, ISOLATED margin)  
**Primary pair:** BTC/USDT (configurable)  
**Scalp timeframe:** 3m (hardcoded, P0-2)  
**Swing timeframe:** 4h  
**Leverage:** Configurable (default 5x, scales down on drawdown)  
**Modes:** Live, Demo (Binance testnet), Paper (local simulation)

---

## 2. Architecture

```
main.py
  ├── MarketState / BinanceWSManager   (WebSocket data feed)
  ├── FeatureCache                     (indicator computation, one-per-candle)
  ├── AlphaEngine                      (17 individual signal votes)
  ├── SignalScoring                    (weighted aggregation → BUY/SELL/HOLD)
  ├── RiskEngine                       (position sizing, SL/TP, kill-switch)
  ├── OrderExecutor                    (Binance Futures order management)
  ├── SwingStrategy                    (4h EMA50/200 + S/R + MTF)
  ├── ScalpStrategy                    (legacy; EMA9/21 + RSI + NW envelope)
  ├── TradeTrackerV2                   (JSONL persistence + signal attribution)
  ├── WeightOptimizer                  (LLM-driven weekly weight updates)
  └── TelegramAlerts                   (all notifications)
```

### Two Execution Modes

**WebSocket mode** (`USE_WEBSOCKET=true`, default):  
- `BinanceWSManager` connects to kline/depth/trade streams  
- `MarketState` holds latest book/price/candle state with change-flags  
- `StateChangeDispatcher` fires callbacks: `candle_complete`, `book_update`, `price_jump`, `book_invalidated`  
- `candle_complete` → triggers full alpha pipeline  

**Polling mode** (`USE_WEBSOCKET=false`, fallback):  
- REST `fetch_ohlcv` call every N seconds  
- Same downstream pipeline

## 3. Signal System (17 votes)

All votes are generated by `AlphaEngine.generate_votes()` from a `FeatureSet`. Each vote is a `Vote(direction, strength, reason)` with direction BUY/SELL/HOLD and strength 0–1.

| Signal | Key | Description |
|--------|-----|-------------|
| EMA Cross | `ema_cross` | EMA9 vs EMA21 crossover (±0.1% deadband) |
| RSI Zone | `rsi_zone` | RSI<35=BUY, RSI>65=SELL |
| MACD Cross | `macd_cross` | MACD line vs signal line |
| BB Bounce | `bb_bounce` | Price at lower/upper Bollinger Band |
| BB Squeeze | `bb_squeeze` | Band compression breakout detection |
| VWAP Cross | `vwap_cross` | Price crosses VWAP |
| OBV Trend | `obv_trend` | On-balance volume direction |
| Volume Spike | `volume_spike` | Volume >= 1.5x rolling average |
| Swing Bias | `swing_bias` | Injected from `SwingStrategy` |
| Funding Bias | `funding_bias` | Negative funding=BUY, positive=SELL (8h cache) |
| MTF Bias | `mtf_bias` | 15m EMA8/20 + RSI14 confirmation |
| NW Signal | `nw_signal` | Nadaraya-Watson envelope cross |
| ADX Filter | `adx_filter` | ADX regime filter |
| OB Imbalance | `ob_imbalance` | Order book bid/ask ratio |
| Trade Aggression | `trade_aggression` | Taker buy/sell ratio from recent trades |
| Liquidity Wall | `liquidity_wall` | Large bid/ask wall within 0.3% |
| Liquidity Sweep | `liquidity_sweep` | Stop-hunt sweep detection (2-candle cooldown) |

### Scoring Formula

```
weighted_score = Σ(vote_i × weight_i)
action = BUY if score >= +3.0 else SELL if score <= -3.0 else HOLD
confidence = min(|score| / 6.0, 1.0)
```

A consensus check also applies: if the dominant direction accounts for <65% of total signal weight, the action is downgraded to HOLD.

## 4. Module Reference

### `config.py`
Loads all configuration from environment variables (`.env` file). Key sections:
- Exchange credentials (`BINANCE_API_KEY`, `BINANCE_SECRET`, `BINANCE_DEMO_TRADING`)
- Telegram (`TELEGRAM_BOT_TOKEN`, `TELEGRAM_CHAT_ID`)
- Core trading params: `SYMBOL`, `TIMEFRAME`, `LEVERAGE`, `LOOKBACK_CANDLES`
- Risk params: `RISK_PER_TRADE`, `DAILY_DRAWDOWN_LIMIT`, `MAX_OPEN_POSITIONS`, `MAX_CONCURRENT_TRADES`
- ATR params: `ATR_SL_MULTIPLIER`, `ATR_TP_MULTIPLIER`, `MIN_REWARD_RISK_RATIO`
- Kelly: `KELLY_ENABLED`, `KELLY_FRACTION`, `KELLY_MAX_FRACTION`, `KELLY_MIN_TRADES`, `KELLY_RAMP_TRADES`
- Trailing stop: `ATR_TRAIL_ENABLED`, `SCALP_TRAIL_ACTIVATE_PCT`, `SCALP_TRAIL_ATR_MULT`
- BB/ADX/Volume: `BB_PERIOD`, `BB_STD`, `ADX_PERIOD`, `ADX_TREND_THRESHOLD`, `ADX_RANGE_THRESHOLD`, `VOL_SPIKE_MULT`
- Regime disabled signals: `DISABLED_SIGNALS_BY_REGIME` dict
- Swing: `SWING_ENABLED`, `SWING_SYMBOLS`, `SWING_TIMEFRAME`, `SWING_LEVERAGE`, `SWING_EMA_FAST/SLOW`, `SWING_STOP_LOSS_PCT`, `SWING_TAKE_PROFIT_PCT`
- LLM optimizer: `LLM_API_URL`, `LLM_API_KEY`, `LLM_MODEL`, `MIN_WEIGHT`, `MAX_WEIGHT`
- WebSocket: `USE_WEBSOCKET`
- Paper trading: `PAPER_TRADING_MODE`, `INITIAL_BALANCE`

### `feature_cache.py`
- `FeatureSet` dataclass — container for all computed indicator values for one candle
- `FeatureCache.update(ohlcv_df)` — computes all indicators once per candle and returns a `FeatureSet`
- `OrderFlowCache` — rolling deque of recent trades (30s window) and book snapshots for OB/aggression signals
- Key fields in `FeatureSet`: `close`, `high`, `low`, `ema_fast`, `ema_slow`, `rsi`, `atr`, `atr_ma50`, `vwap`, `bb_upper/lower/width`, `macd_line/signal`, `obv`, `adx`, `volume`, `volume_ma`, `nw_long_cross/nw_short_cross`, `regime`, `ob_imbalance`, `trade_aggression_ratio`, `liquidity_sweep_bull/bear`

### `alpha_engine.py`
- `Vote` dataclass: `direction` (BUY/SELL/HOLD), `strength` (0–1), `reason`
- `AlphaVotes` dataclass: one `Vote` field per signal (17 total)
- `AlphaEngine.generate_votes(features, funding_rate, swing_vote, mtf_vote)` — produces all votes from a FeatureSet
- `AlphaEngine.generate_votes_with_funding(features, symbol, ...)` — async version that fetches live funding rate
- `FundingRateCache` — TTL cache (8h) for funding rates to avoid repeated API calls
- Liquidity sweep has a 2-candle cooldown to avoid double-firing

### `signal_scoring.py`
- `SignalScoring.score(votes, features)` → `ScoringResult`
- `ScoringResult` fields: `action`, `score`, `abs_score`, `confidence`, `threshold`, `vote_details`, `weight_details`, `weighted_breakdown`, `contributing_signals`, `regime`, `volatility_filter_triggered`
- Volatility filter (P1-3): returns HOLD if `atr / atr_ma50` is outside `[ATR_RATIO_MIN, ATR_RATIO_MAX]`
- Regime signal disabling (P1-4): zero-weights signals listed in `DISABLED_SIGNALS_BY_REGIME[regime]`
- Weights loaded from `weights.json` (flat or per-regime format)
- `DEFAULT_WEIGHTS` (updated 2026-03-14): ema_cross=1.4, rsi_zone=1.2, macd_cross=1.2, bb_bounce=1.0, bb_squeeze=1.3, vwap_cross=1.1, obv_trend=0.9, volume_spike=1.0, swing_bias=1.6, nw_signal=1.2, adx_filter=1.0, funding_bias=0.8, mtf_bias=1.5, ob_imbalance=1.1, trade_aggression=1.3, liquidity_wall=0.8, liquidity_sweep=1.7

### `risk_engine.py`
- `RiskEngine.__init__(exchange)` — initialises with config; calls `_sync_daily_balance()`
- **Kill switch**: `check_kill_switch()` — trips if daily drawdown >= `DAILY_DRAWDOWN_LIMIT` (default 3%)
- **Circuit breaker**: `check_circuit_breaker()` / `record_trade_pnl()` — trips if realized daily loss >= `DAILY_LOSS_LIMIT`
- **Position sizing**: `calculate_position_size(entry, stop, kelly_fraction)` — fixed fractional with optional Kelly warm-up ramp
- **Effective leverage**: `get_effective_leverage()` — scales down 2x at 2% drawdown, 4x at 3.5%+
- **SL/TP**: `get_stop_loss()` / `get_take_profit()` — ATR-based with R:R check, regime-aware multipliers (TRENDING widens SL 20%, TP 30%), falls back to percentage
- **Trailing stop**: `init_trailing_stop()`, `update_trailing_stop()`, `check_trailing_stop_hit()`, `remove_trailing_stop()` — ATR-based ratcheting trail, activates after `SCALP_TRAIL_ACTIVATE_PCT` profit
- **Concurrent trade limiter**: `check_total_concurrent_trades()` — counts all open positions (scalp + swing)
- **Swing methods**: `get_swing_stop_loss()`, `get_swing_take_profit()`, `get_swing_trailing_stop()`, `calculate_swing_position_size()`, `check_swing_max_positions()`, `check_swing_total_exposure()`
- **Pre-trade gate**: `can_open_trade()` — runs all checks in sequence (kill switch → circuit breaker → max positions → concurrent limit)
- **Sanity guard**: `sanity_guard(entry, vwap, ask, bid, last_trade_time, size)` — checks VWAP deviation, spread, data freshness, and clamps position size
- **Daily reset**: `reset_daily()` — resets all daily counters at UTC midnight
- **Balance cache**: 3-second TTL to avoid repeated API calls; falls back to cached value on failure

### `order_executor.py`
- `OrderExecutor.__init__(exchange, risk_engine)`
- `open_position(symbol, side, amount, stop_loss, take_profit, expected_entry)` — full flow:
  1. Spread guard check
  2. Market entry order
  3. Slippage check — emergency flatten if exceeded
  4. Recalculate SL/TP from actual fill price
  5. Atomic bracket (SL + TP); cancels SL and flattens if TP fails
- `close_position(symbol)` — fetches position info, cancels bracket orders, places reduce-only market order
- `check_spread(symbol)` — fetches order book, computes bid-ask spread as % of mid price, blocks if > `MAX_SPREAD_PCT`
- `set_leverage(symbol, leverage)` / `set_margin_type(symbol)` — Binance Futures setup
- `cancel_all_orders(symbol)` — cancels all open bracket orders for a symbol
- `get_open_positions(symbol)` / `get_position_info(symbol)` — position queries
- `reconcile_position(trade_tracker, telegram_alerts)` — on startup, restores any existing open position into TradeTracker
- `FatalExchangeError` — raised on auth/IP-ban errors; main loop shuts down on this
- Error classification: `_classify_error()` — `fatal` (auth/ban) vs `transient` (network/rate limit)
- Paper trading mode: all order methods return mock responses

### `trade_tracker_v2.py`
- `TradeTrackerV2` — persists all trades to a JSONL file (one JSON object per line)
- `record_trade(symbol, side, trade_type, entry, exit, size, reason, entry_time, scoring)` — calculates P&L with 0.1% taker fee per side, persists, updates streaks and signal stats
- `get_cumulative_stats()` — all-time stats: win rate, total P&L, profit factor, avg win/loss, streaks, EV, per-signal performance
- `get_session_stats()` / `get_daily_stats()` — filtered views
- `get_ev()` — EV per trade = (win_rate × avg_win) − (loss_rate × avg_loss)
- `get_signal_performance()` — per-signal win rate, avg P&L, total P&L (used by WeightOptimizer)
- `restore_open_position()` — called by reconcile on restart
- Streak tracking: `_current_streak` (positive=wins, negative=losses), `_longest_win/lose_streak`

### `weight_optimizer.py`
- `WeightOptimizer(tracker, weights_file)` — LLM-driven agentic feedback loop
- `run_optimization_cycle()` — full flow:
  1. Profit factor gate: freeze weights if profit factor < 1.2 and ≥10 trades
  2. Fetch signal performance from TradeTrackerV2
  3. Build prompt with performance data and boundaries
  4. POST to LLM API (`LLM_API_URL` with Bearer auth)
  5. Parse JSON response, sanitize weights (clamp to `[MIN_WEIGHT, MAX_WEIGHT]`)
  6. Atomic file write via temp file + rename
- `_validate_and_sanitize(weights)` — safety rail: enforces per-signal bounds, falls back to defaults for missing signals
- `_check_profit_factor_gate()` — sends Telegram alert if weights frozen
- Optimization runs weekly (triggered by main loop at 30-trade milestone)
- Temperature set to 0.1 for deterministic LLM output

### `telegram_alerts.py`
- `TelegramAlerts` — async sender with 1-second rate limit between messages
- Falls back to plain text if Telegram rejects HTML (400 error)
- Key methods:
  - `send_trade_alert(side, symbol, entry, sl, tp, size, leverage, ...)` — enriched entry alert with volume, regime, BB squeeze, Kelly fraction, ATR, confidence
  - `send_close_alert(side, symbol, entry, exit, pnl, pnl_pct, ...)` — exit alert with trailing stop info, hold duration
  - `send_trailing_stop_alert(...)` — ATR trail activation
  - `send_squeeze_alert(...)` — BB squeeze breakout
  - `send_regime_change_alert(...)` — market regime shift
  - `send_max_trades_alert(...)` — concurrent limit hit
  - `send_circuit_breaker_alert(...)` — daily loss limit
  - `send_kill_switch_alert()` — daily drawdown limit
  - `send_stats(session, cumulative, ...)` — full P&L dashboard
  - `send_daily_summary(...)` — EOD report with premium metrics
  - `send_heartbeat(...)` — periodic alive check with uptime, last signal, regime, spread
  - `send_startup_message()` — bot start with feature status
  - `send_shutdown_message(reason)` — bot stop
  - `send_error_alert(error)` — error notification
  - `get_updates(offset, timeout)` — polls Telegram for `/stats` commands
  - `send_swing_trade_alert(...)` — swing entry alert
- Mode tag: PAPER (local sim), DEMO (Binance testnet), LIVE (real)

### `swing_strategy.py`
- `SwingStrategy` — 4h EMA50/200 strategy
- `generate_signal(df_4h, df_15m)` → `SwingTradeSignal`
- Signal logic: EMA50/200 cross + RSI zone filter (40-50 for longs, 50-60 for shorts) + S/R from pivot highs/lows (window=5) + volume confirmation + ATR
- MTF confirmation (P1-8): 15m EMA8/20 + RSI14, cached for 15 minutes
- `SwingTradeSignal` fields: `signal`, `confidence`, `entry_price`, `symbol`, `ema_fast/slow`, `rsi`, `support`, `resistance`, `reason`, `atr`, `mtf_vote`

### `strategy.py` (ScalpStrategy — legacy)
- Original EMA9/21 + RSI + Nadaraya-Watson envelope strategy
- Still used as a pre-filter or standalone fallback
- `generate_signal(df)` → `Signal` enum (BUY/SELL/HOLD)

## 5. Risk Management

| Guard | Threshold | Effect |
|-------|-----------|--------|
| Kill switch | 3% daily drawdown | All trading halted until midnight UTC |
| Circuit breaker | `DAILY_LOSS_LIMIT` realized loss | New entries blocked |
| Leverage scaling | 2%/3.5% drawdown | Leverage halved/quartered |
| Spread guard | `MAX_SPREAD_PCT` | Trade aborted |
| Slippage check | `SLIPPAGE_TOLERANCE` | Emergency flatten if exceeded |
| Max positions | `MAX_OPEN_POSITIONS` | Per-strategy cap |
| Max concurrent | `MAX_CONCURRENT_TRADES` | Combined scalp+swing cap |
| Profit factor gate | PF < 1.2 | Weight optimization frozen |
| ATR trailing stop | `SCALP_TRAIL_ACTIVATE_PCT` | Ratchets SL after profit threshold |
| Sanity guard | VWAP deviation, spread, data freshness | Pre-trade block |

## 6. Trade Flow (Scalp)

```
1. Candle closes (WebSocket kline event OR polling)
2. FeatureCache.update(df) → FeatureSet
3. AlphaEngine.generate_votes_with_funding(features, symbol, swing_vote, mtf_vote) → AlphaVotes
4. SignalScoring.score(votes, features) → ScoringResult
5. If action != HOLD:
   a. RiskEngine.can_open_trade() — all risk gate checks
   b. RiskEngine.get_stop_loss() / get_take_profit() — ATR or %-based
   c. RiskEngine.calculate_position_size(entry, stop, kelly) — size
   d. OrderExecutor.open_position(symbol, side, size, sl, tp, expected)
   e. RiskEngine.init_trailing_stop(order_id, entry, side, atr)
   f. TradeTrackerV2 — registers open trade
   g. TelegramAlerts.send_trade_alert(...)
6. Position monitor loop:
   a. RiskEngine.update_trailing_stop() per candle
   b. If trailing stop hit or SL/TP filled:
      - OrderExecutor.close_position()
      - TradeTrackerV2.record_trade(scoring=...)
      - RiskEngine.record_trade_pnl()
      - TelegramAlerts.send_close_alert()
7. WeightOptimizer triggers after 30 trades (weekly cycle)
```

## 8. Market Regimes

Regimes are computed by `FeatureCache` using ADX:
- `TRENDING_UP` / `TRENDING_DOWN` — ADX > `ADX_TREND_THRESHOLD` (default 25)
- `VOLATILE` — ADX > threshold + high ATR ratio
- `RANGING` / `NEUTRAL` — ADX < `ADX_RANGE_THRESHOLD` (default 20)

Regime-specific signal disabling is defined in `DISABLED_SIGNALS_BY_REGIME` config. Risk engine applies regime multipliers to SL (×1.2 in trending/volatile) and TP (×1.3).

## 10. Key Design Decisions

- **One FeatureSet per candle**: All indicators computed once in `FeatureCache`, shared across all engines. Prevents drift and duplicate calculations.
- **Vote-based signal fusion**: Each signal votes independently; weights allow LLM optimization without changing logic.
- **Atomic bracket orders**: SL+TP placed together; if either fails, position is flattened immediately.
- **Slippage backstop**: After every market fill, actual fill price is compared to expected. Emergency flatten on excess slippage.
- **Error classification**: Auth/ban errors → `FatalExchangeError` → shutdown. Network/rate-limit → log and retry.
- **Paper trading mode**: All exchange calls are intercepted; no real orders placed. Uses `INITIAL_BALANCE` + `daily_realized_pnl` for balance.
- **JSONL trade log**: Append-only; O(1) write per trade; readable without full JSON parse.
- **LLM optimizer safety**: Clamps all weights to `[MIN_WEIGHT, MAX_WEIGHT]`, falls back to defaults for missing signals, freezes if profit factor < 1.2.

## 12. Adding a New Signal

1. Add a field to `AlphaVotes` in `alpha_engine.py` (default `Vote("HOLD", 0.0, "reason")`)
2. Add the vote logic in `AlphaEngine.generate_votes()`
3. Add the signal to `DEFAULT_WEIGHTS` in `signal_scoring.py`
4. Add the vote key in `AlphaVotes.as_dict()` and `get_all_votes()`
5. Optionally add to `DISABLED_SIGNALS_BY_REGIME` in config if regime-specific
6. Restart — `weights.json` will auto-update with the new default weight
